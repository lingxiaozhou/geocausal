1 1. Obtaining observed densities

Now that we have all the required data, let’s move on to obtain the observed densities. The observed densities of treatment (in our example, airstrikes) will eventually be used to estimate causal effects.

The function that we use here is get_obs_density. This function has the following arguments.

  • hfr: The hyperframe that we want to use
  • indep_var: The variable of interest (i.e., treatment)
  • dep_var: A vector of names of dependent variables
  • ngrid: The number of grids (by default, 100)
  • window: The window

get_obs_density function does the following tasks.

  1. Fit an inhomogeneous Poisson model of treatment ~ rhs_list;
  2. Obtain intensities λλ of the entire window by segmenting windows into n=ngridn=ngrid grid cells;
  3. Integrate λλ over the entire window, which is equivalent to the estimate number of outcome events for each time period; and
  4. Calculate the sum of log intensities of all observations for each time period.

The output of this function is a list of the following.

  • independent_variables: A vector of the names of RHS variables
  • coefficients: Coefficients of the model
  • intensity_grid_cells: Intensities as images for each time period
  • estimated_counts: Integrated intensities, which is the number of estimated outcome events for each time period
  • sum_log_intensity: The sum of log intensities of all observations for each time period

For the estimation of causal events, we will use sum_log_intensity, but estimated_counts and intensity_grid_cells are also useful to evaluate how well our model estimates the number of outcome events.

Let’s use the function get_obs_density. Since estimating the intensities over the entire window is computationally demanding, we use ngrid=100ngrid=100.

Again, the output is a list of five elements that are summarized above. To see how each estimated intensities look like visually, we use plot function and select the date of interest. For example, if we are interested in the intensities of the first date of the data, then we can do the following. (We truncate the first 30 days from the original data, which is why the row ID is 31.)

plot(obs_density$intensity_grid_cells[1])

Also, we can compare the estimated and actual count measures by plotting them.

In the original hyperframe dat_hfr, the column Airstrike is a list of ppp objects. In ppp objects, the number of points is saved as n. Also, the predicted number of airstrikes is saved as estimated_counts of the output obs_density. Hence, we can do the following to compare the actual and predicted number of airstrikes.

# Actual count
actual_counts <- unlist(purrr::map(dat_hfr$Airstrike, function(x) x$n))

# Predicted count
predicted_counts <- obs_density$estimated_counts

plot_compare <- data.frame(days = c(31:499),
                           actual_counts, predicted_counts) %>%
  ggplot() +
  geom_line(aes(x = days, y = actual_counts), color = "darkgray") +
  geom_line(aes(x = days, y = predicted_counts), color = "red") +
  theme_bw() +
  annotate("text", label = "Actual", x = 150, y = 50, size = 5, color = "darkgray") +
  annotate("text", label = "Predicted\n(ngrid = 100)", x = 400, y = 70, size = 5, color = "red") +
  labs(title = "Actual vs. predicted counts of airstrikes", x = "Days", y = "Count")

2 Out-of-sample prediction

Now, let’s turn to do some out-of-sample prediction. In this vignette, we do the following experiments.

  1. Dividing into test and training set and fitting the model to the training set
  2. Using the entire data to fit the model, and then applying it to a subset of the data (eg, Baghdad)

To do this, we use a function called predict_obs_density. This function is basically the same as get_obs_density function, except for the fact that now we have ratio argument.

2.1 Using a subset of data to fit the model

When you want to use a fraction of data as a training set, then you set the ratio argument of predict_obs_density. For example, if you set ratio = 0.8, then it uses the first 80% of observations as a training set and fits the model.

We use 70% and 80% of the data (first 70% and 80% observations) as training sets. Since we have a spike in the dataset, the 70% one fails to perform out-of-sample prediction.

# 80% training
out_obs_density_8 <- predict_obs_density(dat_hfr, ratio = 0.8, "Airstrike", 
                                         rhs_var, ngrid = 100, iraq_window)

2.2 Using the entire data to fit the model, and then integrate over a subset of the data

Now, let’s use the fitted model to predict the number of airstrikes in a sub-region. In this case, we continue to use predict_obs_density function, but this time set ratio = 1 and set window = windows$Baghdad. That is, we use the entire dataset to fit the model, and then predict the counts in a window, this time Baghdad and outside of Baghdad. The window objects of Baghdad and outside of Baghdad are saved as windows$Baghdad and windows$outside_Baghdad, respectively.

Let’s take a look at Baghdad first.

# Baghdad
obs_density_baghdad <- predict_obs_density(dat_hfr, ratio = 1, "Airstrike", 
                                           rhs_var, ngrid = 100, window = windows$Baghdad)
#> Fitting the model...
#> Calculating the intensity...
#> Integrating the intensity to obtain the propensity score...
predicted_baghdad <- obs_density_baghdad$estimated_counts

## Calculate the actual count
airstrikes <- dat_hfr$Airstrike

for(i in 1:nrow(dat_hfr)){
  spatstat.geom::Window(airstrikes[[i]]) <- windows$Baghdad
}

actual_baghdad <- unlist(purrr::map(airstrikes, function(x) x$n))

# Plot (Baghdad)
